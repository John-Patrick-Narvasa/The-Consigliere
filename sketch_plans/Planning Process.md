
**Needs**

- BOOK PDFs (Text-heavy strategy/philosophy: _Art of War, Atomic Habits, Building a Second Brain_)
- API 1 core model (e.g., Gemini — excellent for long context, system reasoning, and it's free/cheap to prototype)
- Memory / Session state management (to remember if the user already asked for advice and is now asking for a system)
- Embedding model
- Vector DB (Must be cloud-hosted to survive serverless environments, e.g., Pinecone or Supabase pgvector)
- PDF Parser capable of layout/metadata extraction (e.g., `PyMuPDF` or `pdfplumber` to grab exact page numbers)
- Chunking with metadata attachment
- Semantic Search (Testing retrieval accuracy before building the AI)
- Prompt engineering (Strict "Consigliere" persona + strict citation rules)
- LangGraph integration (Routing logic: Advice, Procedure, System)
- Testing Vector DB & LangGraph (Evaluating citation accuracy and actionability)
- UI via Streamlit (for prototype)


**Want**

- Specific paragraph text verification and linking (Future phase)
- Visual diagram generation for "Systems" (Using Mermaid.js syntax generated by the LLM)
- UI/UX via React and TypeScript
- API via FastAPI
- Deployment: Backend on Render or Railway (FastAPI needs a continuously running server, Vercel will time out), Frontend on Streamlit Cloud or Vercel (if migrating to React).

**Initial Tech Stack**

- **Model:** Gemini
- **Embedder:** HuggingFace (`all-MiniLM-L6-v2`) or OpenAI (`text-embedding-3-small`)
- **LangGraph / LangChain** (For routing and memory)
- **VectorDB:** Pinecone (Free tier is generous and scales easily without local storage issues)
- **Parser:** `PyMuPDF` (Fitz)
- **Backend:** FastAPI hosted on Render/Railway
- **Frontend:** Streamlit


**Storyboard logic process (user to agent)**

- User chats/queries AI agent (Session ID loads previous chat history)
- Embed user query -> Semantic search in Vector DB
- Fetch top matching text chunks _along with their metadata (Book Title, Page #)_
- LangGraph receives: User Query + Retrieved Chunks + Chat History
- LangGraph Router analyzes intent (Does the user want Advice, a Step-by-Step execution plan, or a System/Strategy?)
- Routes to the specific Prompt Node:

  - _Advice Node:_ Wise, straightforward insights tied to the text.
  - _Procedure Node:_ Breaks the text down into atomic, actionable steps.
  - _System Node:_ Synthesizes a mental model or workflow (can output Mermaid.js code for visuals).
- Generate output using the "Consigliere" persona, appending strict citations `[Book Name, Page X]` based ONLY on the retrieved chunks.


**Storyboard data to agent (data to agent) — _[Corrected Order]_**

- Import book PDFs
- Parse PDF input (Extract raw text + exact Page Numbers)
- Chunking (Text Splitter) -> _Crucial step: inject `{"source": book_name, "page": page_number}` into every single chunk's metadata_
- Embed the chunks into vectors [w/ confirmation]
- Upload to Cloud Vector Store (Pinecone/Supabase) [w/ confirmation]
- Run semantic search test script [w/ metrics validating that the correct page is retrieved]


**Step by step process**

**1.Gather Data:**

Find clean PDF books (Art of War, Atomic Habits, etc.) and verify their formatting is readable by scripts.

**2.Build Data Pipeline:**

Write the ingestion script: Parse PDF -> Chunk Text -> Attach Page Metadata -> Embed -> Push to Vector DB.

**3.Test Retrieval (Evaluation Phase):**

Write a simple headless script. Ask 10 test queries, fetch the vectors, and manually verify if the DB returns the correct page numbers. _Do not proceed until this is highly accurate._

**4.Establish the Persona:**

Draft the System Prompts for the "Consigliere" and the strict rules for formatting citations.

**5.Architect the Router (LangGraph):**

Define your Nodes (Advice, Step-by-Step, System) and build the conditional routing logic based on user intent.

**6.Integrate Memory:**

Add session state management to LangGraph so the agent remembers the context of the current conversation.

**7.Build Prototype UI:**

Connect the LangGraph backend to a Streamlit frontend with a chat interface.
